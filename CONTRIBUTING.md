# Contributing to Fountain

Thank you for your interest in contributing to Fountain! Our mission is to promote literature and increase the reading population through a beautiful, non-commercial platform.

## Ways to Contribute

### 1. Authors & Publishers: Content Updates
We highly value the accuracy of the information presented on our platform. If you are an author or publisher featured on Fountain:
- Use the [Content Update Request](https://github.com/michaelwilkins/fountain/issues/new?template=content-update.yml) template.
- Specify the changes needed (images, quotes, SNS links, etc.).
- Our team (or AI agents) will process these requests directly into the repository.

### 2. Suggesting New Authors
If there is a novelist you believe should be featured:
- Use the [New Author Suggestion](https://github.com/michaelwilkins/fountain/issues/new?template=new-author.yml) template.
- Provide as much detail as possible, including links to their works and official sites.

### 3. Technical Contributions
We welcome developers to help improve the app's performance, design, and accessibility.
- **Bug Reports:** Use the [Bug Report](https://github.com/michaelwilkins/fountain/issues/new?template=bug-report.yml) template.
- **Pull Requests:** 
    - Fork the repository.
    - Create a new branch for your feature or fix.
    - Ensure your code follows the existing style and conventions.
    - Submit a PR using our [PR Template](.github/pull_request_template.md).

## Development Setup

1. **Clone the repo:**
   ```bash
   git clone https://github.com/michaelwilkins/fountain.git
   ```
2. **Install dependencies:**
   ```bash
   pnpm install
   ```
3. **Start development server:**
   ```bash
   pnpm dev
   ```

## Licensing

By contributing to Fountain, you agree that your contributions will be licensed under the **CC BY-ND 4.0** license. All content contributions must respect the "No-Derivatives" clause to preserve the integrity of the authors' original works and intent.
